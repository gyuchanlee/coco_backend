## 1. 한 줄 정의

**경북 지역 소규모 여행자(1~4인)를 위한 추천 & 예산 스마트 플래너.**

인원수에 맞춘 큐레이션으로 숙소·식당·관광지를 추천하고, 코스를 구성하는 즉시 총예산과 1인당 부담액을 실시간 산출하며, 완성된 일정과 비용 분담 내역을 카카오톡으로 한 번에 공유한다.

## 2. 배경 및 문제

| # | 문제 | 근거 |
| --- | --- | --- |
| P1 | **소규모 여행 니즈 미충족** | 1인 가구 증가·개인화로 혼행/2~3인 소그룹이 주류가 되었으나, 기존 플랫폼은 4인 이상·가족 단위 정보에 편중. |
| P2 | **예산 예측의 번거로움** | 고물가·가성비 중심 소비. 숙박·식비·교통·입장료를 일일이 조사·합산해 예산을 예측하기 어렵다. |
| P3 | **여행 계획 피로감** | 블로그·SNS·예약앱·지도앱을 오가며 정보를 교차 검증하고, 동행인과 코스 공유·더치페이를 논의하는 과정의 마찰. |

## 3. 타깃 사용자 & 핵심 시나리오

- **주 타깃**: 가성비·계획 효율을 중시하는 2030 세대 소규모 여행자(1~4인).
- **핵심 시나리오 (해피 패스)**
  1. 메인(`/`)에서 목적지·일정·인원·테마를 한 줄로 선택해 검색한다. → 조건이 URL 쿼리로 `/planner`에 전달된다.
  2. 플래너(`/planner`)에서 인원수에 맞게 큐레이션된 스팟을 **지도/리스트 토글**로 본다.
  3. POI를 클릭하면 **우측 상세 드로어**로 설명·운영시간·요금·사진을 확인하고 `Day N`의 '내 코스'에 추가한다.
  4. 코스에 스팟을 추가/삭제하거나 POI별 금액을 인라인 수정할 때마다 **하단 예산 대시보드**의 총예산·1인당 금액이 실시간 갱신된다.
  5. 완성된 코스를 **카카오톡/링크로 공유**한다. → 수신자는 읽기전용 뷰어(`/share/:id`)로 일정·비용 분담을 본다.
  6. 게스트가 저장/공유를 누르면 **로그인 모달**이 뜨고, 로그인 후 작업을 이어서 컬렉션(`/collection`)에 저장한다.
  7. 컬렉션의 저장 코스를 다시 열면 플래너(`/planner?load=:id`)로 불러와 재편집한다.

## 4. 핵심 기능

제안서의 3대 기능 축을 그대로 채택한다. 각 기능을 화면/라우트/스토어/API 계층에 매핑한다 (상세는 [§10 화면 상세 명세](#10-화면-상세-명세)·[§13 구현 매핑](#13-구현-매핑-코드베이스--화면)).

### F1. Who & How Many — 인원수 기반 스마트 큐레이션
- 여행 인원(**1인 / 2인 / 3~4인**)과 일정을 선택하면 해당 인원수에 적합한 경북 내 숙소·식당·관광지가 맞춤 노출.
- 예: 혼자 먹기 좋은 바테이블 식당(1인), 감성 숙소(2인), 비용 나누기 좋은 패밀리룸(3~4인).
- 입력 모델: 자유 카운터(`number`) → 추천 시 `n→버킷` 매핑(`1→1`, `2→2`, `≥3→'3-4'`). 예산 1인당 분배는 실제 `n` 사용 (OQ2).
- **연결 화면**: 메인 검색(S1, `/`) → 플래너(S2, `/planner`).

### F2. Smart Itinerary & Budget Planner — 실시간 예산 계산기 & 코스 빌더
- **단일 워크스페이스**: 한 화면(`/planner`)에 좌측 결과(지도/리스트 토글) · 우측 코스 패널 · 하단 예산 대시보드를 배치.
- **기본 추천 코스**: 우측 패널에 기본 일정 표시, 커스텀 또는 새 코스 생성.
- **코스 빌더**: 지도/리스트의 POI를 클릭 → 상세 드로어 → `Day N`의 '내 코스'에 추가/삭제.
- **Day별 일정**: 일정 길이(N박)에서 Day 수를 도출, 코스를 `Day1/Day2/…` 탭·섹션으로 구분. 예산은 Day 합산 → 전체 합산.
- **시간대 반영**: 박물관·식당 등 운영 시간을 반영, 불가능한 시간대 코스는 선택 불가 표시.
- **실시간 예산 계산**: 장소별 평균 객단가(숙박비·식사비·입장료)를 기반으로 총예상 예산과 1인당 부담액을 하단 대시보드에서 실시간 업데이트. **POI별 금액 인라인 수정** 가능(기본값=백엔드 근사치, override는 코스에 저장).
- **경로 시각화**: 코스 순서대로 이동 경로·지도 자동 표시 (카카오지도 API로 경로·POI 시각화).
- **연결 화면**: 플래너(S2, `/planner`, `Planner.tsx`) + POI 상세 드로어(S2a) + 로그인 모달(S2b).

### F3. One-Click Share — 카카오톡 기반 비용 분담 공유
- 완성된 여행 일정 + 상세 예산 내역(항목별 금액, 1인당 송금 요망 금액)을 카카오톡 또는 링크로 즉시 공유.
- 카카오톡 공유하기 API로 간단 공유.
- 1인당 송금액은 실제 `n` 균등 분배(기본). 사용자별 차등 분배는 향후 확장(OQ7).
- **연결 화면**: 플래너 공유 액션(S2) → 읽기전용 공유 뷰어(S3, `/share/:id`). 수신자는 '나도 편집하기' CTA로 플래너에 유입.

## 6. 데이터 활용 (한국관광공사 OpenAPI — 필수)

활용 API: **국문 관광정보 조회 서비스** — `areaBasedList`(지역기반 조회), `detailCommon`(공통정보), `detailIntro`(소개정보), `detailInfo`(상세정보).

> 아래 수집·가공은 모두 **백엔드 책임**이다. 이 FE repo는 가공된 결과를 백엔드 API로 받아 표시한다.

| 단계 | 방식 |
| --- | --- |
| ① 지역 필터링·기초 수집 | `areaBasedList`에 `areaCode=35`(경상북도) + `sigunguCode`를 파라미터로, contentTypeId별 수집: 관광지(12)·문화시설(14)·숙박(32)·음식점(39). 안정적 응답을 위해 서비스 DB에 적재, **정합성 위해 월 1회 주기 수집**. |
| ② 인원·예산 메타데이터 생성 | 수집한 POI의 콘텐츠 ID로 상세정보 API 호출. **음식점**: 외식통계조회·한국관광 데이터랩 '지역별 관광소비' 데이터로 평균 식비 근사치 제공, 카카오맵 링크로 사용자 수정 가능. **숙박**: 객실 수용 인원·비수기/성수기 요금 분석 → 1·2·4인 적합 숙소 분류, 예산 계산기 기본값. |
| ③ 동선 최적화·교통비 추정 | DB의 `MapX`/`MapY` 좌표로 선택 장소 간 직선/예상 이동거리 계산 → 대략적 유류비·대중교통비를 예산에 자동 합산. |

## 7. 지역 특화 테마 (경북)

- **[경주]** 나홀로 도보 역사 산책 & 황리단길 혼밥 투어
- **[포항/영덕]** 2~3인 해안도로 드라이브 & 오션뷰 가성비 숙소
- **[안동]** 4인 친구 모임 전통 한옥 고택 체험 & 찜닭 거리 투어
- **로컬 상권 연계**: 소규모 식당·카페 정보 제공으로 지역 경제 선순환 유도.

## 8. 발전 방향 & 기대효과

**발전 단계**
- **[단기] UGC 데이터 축적** — 공공 API 의존에서 시작, 사용자가 만든 '가성비 실전 코스'·'실제 지출 비용'을 리뷰로 축적해 추천 로직 정확도 고도화.
- **[중기] 소상공인 제휴 할인** — 추천 골목 상권 식당·카페와 제휴, 앱 내 전용 쿠폰 발급 + 수수료 기반 BM.
- **[장기] B2B/B2G 확장** — 경북 실증 모델을 전국 확장, '소규모 여행객 이동·소비 동선 데이터'를 비식별화해 지자체 관광 정책용으로 제공.

**기대효과**
- *경제적*: 예산 투명성으로 2030 경북 관광 진입 장벽 완화, 소비를 골목상권으로 분산.
- *사회적*: 계획·정산 마찰과 피로감 감소 → 만족도 높은 여행 경험.
- *공공데이터 가치*: 단순 관광지 정보를 '실시간 예산 자동 계산' 유틸리티와 결합해 고부가가치 정보로 재창출.

## 9. 화면 인벤토리 (Information Architecture)

> 백엔드 통합 PRD·기능명세·API명세의 기준이 되는 화면 목록. `S*`는 본 문서 내 화면 식별자다.
> 인증 정책은 하이브리드(OQ6): 탐색·플래닝은 게스트 허용, **저장·공유 시점에만 로그인** 유도.

| ID | 화면 | 라우트 | 유형 | 인증 | 핵심 목적 |
| --- | --- | --- | --- | --- | --- |
| S1 | 메인 / 검색 | `/` | 페이지(랜딩) | 게스트 | 목적지·일정·인원·테마 입력 → 플래너 진입 |
| S2 | 플래너 워크스페이스 | `/planner` | 페이지 | 게스트(저장 시 로그인) | 결과 탐색 + 코스 빌더 + 실시간 예산 |
| S2a | POI 상세 드로어 | (S2 내부 오버레이) | 드로어/모달 | 게스트 | POI 상세정보 확인 + 코스 추가 |
| S2b | 로그인 모달 | (전역 오버레이) | 모달 | — | 게이팅 시점 카카오 로그인 + 작업 이어하기 |
| S3 | 공유 뷰어 | `/share/:id` | 페이지 | 공개(읽기전용) | 수신자가 일정·비용분담 확인, 편집 유도 |
| S4 | 컬렉션(마이페이지) | `/collection` | 페이지 | 로그인 | 저장 코스 목록·관리, 프로필 진입점 |

## 10. 화면 상세 명세

> 각 화면의 구성요소·상태(states)·백엔드 요구(데이터/액션)를 정의한다. API 엔드포인트 시그니처는 백엔드가 API 명세서에서 확정(여기선 **요구사항**만 기술).

### S1. 메인 / 검색 (`/`)
- **구성요소**: 히어로 카피, 한 줄 검색바 — 목적지(멀티셀렉트, 경북 23개 시군구), 일정(날짜 범위 피커), 인원(카운터), 테마(멀티셀렉트), 검색 버튼.
- **동작**: 검색 → 조건을 URL 쿼리로 직렬화해 `/planner`로 이동(`?dest=…&start=…&end=…&pax=…&theme=…`). 동시에 `searchStore`(Zustand)에 보존.
- **상태**: 기본 / 입력 검증(목적지·일정 미선택 시 안내) / 데이터 미비 목적지 "준비 중" 비활성 표시(OQ3 — 경주·포항·영덕·안동 외).
- **백엔드 요구**: 시군구 목록·테마 목록(또는 FE 상수). 데이터 보유 시군구 플래그(준비 중 표시용).

### S2. 플래너 워크스페이스 (`/planner`)
- **레이아웃**: 데스크톱 3분할(좌 결과 / 우 코스 / 하 예산), 모바일은 탭·시트 전환.
- **좌측 — 결과 영역**: 지도(카카오맵, POI 마커) ↔ 리스트 토글. 인원 버킷·테마·시군구 필터 결과. POI 카드(썸네일·이름·카테고리·예상금액).
- **우측 — 내 코스 패널**: `Day1/Day2/…` 탭. Day별 POI 순서 리스트(드래그 정렬), 추가/삭제. 코스 순서대로 지도 경로 표시. 운영시간 충돌 시 경고/선택불가.
- **하단 — 예산 대시보드**: 총예산, 1인당 부담액, 카테고리별(숙박·식비·교통·입장) 소계. **POI별 금액 인라인 수정** → 즉시 재집계. 교통비는 백엔드 추정값 표시(FE 계산 안 함).
- **액션 바**: 저장(→ 게스트면 S2b), 공유(→ S2b 후 링크/카카오), 새 코스.
- **진입**: S1 검색 결과 / `?load=:id`(컬렉션 재편집) / `?share=…` 없음(공유는 S3 전용).
- **상태**: 로딩(스켈레톤) / 결과 없음(빈 상태) / 데이터 준비 중 시군구 / API 오류(재시도) / 저장 성공 토스트.
- **상태 전달**: URL 쿼리 + `searchStore`/`courseStore`/`budgetStore`(Zustand).
- **백엔드 요구**:
  - 검색 조건(시군구·인원버킷·테마·기간)으로 큐레이션된 POI 목록(좌표·카테고리·썸네일·예상 객단가 포함).
  - POI 상세(설명·운영시간·요금·사진·카카오맵 링크) — S2a용.
  - 기본 추천 코스(시군구·인원·일수 기반).
  - 코스 좌표열에 대한 경로/이동거리·교통비 추정.
  - 코스 저장/수정/삭제(로그인 사용자), `load=:id` 조회.

### S2a. POI 상세 드로어 (S2 내부 오버레이)
- **트리거**: 좌측 결과(지도 마커/리스트 카드) 클릭. 라우팅 없이 상태로 제어(데스크톱=우측 드로어, 모바일=바텀시트).
- **구성요소**: 사진, 이름·카테고리, 설명(detailCommon), 운영시간·휴무(detailIntro), 요금/예상 객단가(detailInfo), 카카오맵 링크, **"내 코스(Day N)에 추가"** 버튼.
- **상태**: 로딩 / 상세 오류 / 이미 코스에 포함됨 표시.
- **백엔드 요구**: `contentId` 기반 상세 조회(공통·소개·상세 통합 응답 권장).

### S2b. 로그인 모달 (전역 오버레이)
- **트리거**: 게스트가 저장·공유 클릭 시. 카카오 로그인 버튼.
- **핵심 UX**: 로그인 성공 후 **직전 작업(저장/공유)을 그대로 이어서 수행**. 편집 중 코스 상태는 보존(비로그인 임시 보관 필요 — OQ8).
- **백엔드 요구**: 카카오 OAuth 콜백·세션/토큰 발급, 현재 사용자 조회.

### S3. 공유 뷰어 (`/share/:id`)
- **목적**: 링크/카카오톡 수신자가 보는 **읽기전용** 화면. 편집 UI 없음.
- **구성요소**: 여행 요약(목적지·기간·인원), Day별 코스 + 지도 경로(읽기전용), 항목별 금액, **1인당 송금 요망 금액**, "나도 편집하기" CTA(→ 코스 복제하여 `/planner`).
- **인증**: 공개(게스트도 열람). 만료/삭제 시 안내.
- **상태**: 로딩 / 만료·없음(404 유사) / 정상.
- **링크 영속성(OQ5)**: 1단계 임시 URL 인코딩(A) → 2단계 서버 저장(B). `id`는 서버 저장 단계의 식별자.
- **백엔드 요구**: 공유 ID로 코스+예산 스냅샷 조회, 공유 생성(저장 코스 → 공유 ID).

### S4. 컬렉션 / 마이페이지 (`/collection`)
- **목적**: 저장 코스 보관함 + 내 영역(별도 마이페이지 없이 겸함).
- **구성요소**: 저장 코스 카드 그리드(썸네일·제목·기간·인원·총예산). 카드 액션 — 열기(`/planner?load=:id`), 공유(→ S3 링크), 삭제. 헤더 드롭다운에 프로필/로그아웃.
- **상태**: 로딩 / 빈 상태(첫 저장 유도 CTA) / 오류.
- **인증**: 로그인 필요(미로그인 시 S2b 또는 `/auth/login`).
- **백엔드 요구**: 사용자별 저장 코스 목록 조회, 삭제, 코스→공유 링크 생성.

### S5/S6. 인증 (`/auth/login`, `/auth/register`)
- 카카오 OAuth 위젯(`KakoLoginComponent`) 존재. 직접 진입 경로. 게이팅은 주로 S2b 모달로 처리, 본 페이지는 보조.
- `returnTo` 지원(로그인 후 원래 위치 복귀).

### S7. 서비스 소개 (`/about`)
- 서비스 가치·핵심 기능·사용법 소개. 공모전 심사위원·신규 사용자 온보딩용. 정적 콘텐츠.

### S8. 404 (`*`)
- `NotFoundLayout`. 메인으로 복귀 CTA.

## 11. 화면 전이 흐름 (Flow)

```
[S1 메인/검색] --검색(쿼리)--> [S2 플래너 워크스페이스]
                                  │  ├─ POI 클릭 ──> [S2a 상세 드로어] --추가--> 내 코스(Day N)
                                  │  ├─ 저장/공유(게스트) ──> [S2b 로그인 모달] --성공--> 작업 이어하기
                                  │  ├─ 저장(로그인) ──> [S4 컬렉션]
                                  │  └─ 공유(로그인) ──> 공유 링크/카카오 ──> [S3 공유 뷰어]
[S3 공유 뷰어] --나도 편집하기(코스 복제)--> [S2 플래너]
[S4 컬렉션] --열기(?load=:id)--> [S2 플래너]
[헤더] --> [S1] / [S4] / [S7 소개] / 로그인(S5)
```

## 12. 전역 UX 규칙 (handoff 기준)

- **반응형**: 데스크톱 3분할 워크스페이스 ↔ 모바일 탭/시트 전환. `Layout`의 반응형 폭 준수.
- **상태 패턴**: 모든 데이터 화면은 *로딩(스켈레톤) / 빈 상태 / 오류(재시도)* 3종을 갖춘다.
- **데이터 커버리지(OQ3)**: 목적지는 23개 시군구 멀티셀렉트 전체 노출, 더미 코스·예산 데이터는 경주·포항·영덕·안동 우선. 나머지는 "준비 중" 빈 상태.
- **인증 게이팅(OQ6)**: 탐색·플래닝 게스트 허용, 저장·공유 클릭 시 S2b 모달, 성공 후 작업 이어하기.
- **상태 관리**: 검색조건은 URL 쿼리 + `searchStore`. 코스·예산은 Zustand 스토어(`courseStore`, `budgetStore`). 새로고침·딥링크·공유 견딤.
- **금액 표기**: 원화, 1인당 = 총액 / 실제 `n` 균등 분배(기본).
- **외부 SDK**: 카카오맵(경로·POI), 카카오 공유, 카카오 로그인. 키는 `.env`(노출 금지).

## 14. 가정 및 미결 (Open Questions)

> 제안서에 명시되지 않았거나 코드베이스와 어긋나는 사항. ✅는 본 PRD에서 확정됨.

- **OQ1. 백엔드/DB 소유 주체** — ✅ 백엔드·DB·수집은 별도 repo/팀, 이 repo는 FE 전용. `src/api/`는 자체 백엔드 API만 소비(관광공사 직접 호출 안 함).
- **OQ2. 인원 입력 모델** — ✅ 카운터(`number`) 유지 + 추천 시 `n→버킷` 매핑(`1→1`, `2→2`, `≥3→'3-4'`), 예산 1인당 분배는 실제 `n`.
- **OQ3. 목적지·테마 옵션** — ✅ 목적지는 경북 23개 시군구 멀티셀렉트 전체 노출, 더미 코스·예산은 경주·포항·영덕·안동 우선(나머지 "준비 중"). 테마는 일반 테마 유지·확장. *(구현 TODO)*
- **OQ4. 예산 데이터 출처 신뢰도** — ✅ UI: POI별 금액 **인라인 수정** 가능(기본값=백엔드 근사치, override는 코스에 저장). 교통비는 FE 계산 없이 백엔드 추정값 표시. 식비·교통비 근사치의 초기 정확도 한계는 추가 통계로 보완. *(데이터 소싱 정책은 백엔드)*
- **OQ5. 공유 링크 영속성** — ✅ 임시 URL 인코딩(A) → 서버 저장(B) 단계적 전환. S3는 `/share/:id`로 서버 저장 단계 기준.
- **OQ6. 인증 게이팅** — ✅ 하이브리드 — 탐색·플래닝 게스트, 저장·공유 시점 S2b 로그인 모달 + 작업 이어하기.
- **OQ7. 비용 분담 방식** — 🔶 기본은 실제 `n` **균등 분배**. 사용자별 차등 분배(누가 더/덜)는 향후 확장. *(백엔드와 데이터 모델 합의 필요)*
- **OQ8. 비로그인 코스 임시 보관** — 🔶 게스트가 편집 중인 코스를 로그인 게이팅·새로고침 동안 어디에 보관할지(로컬스토리지 vs 서버 임시). 1단계 로컬스토리지 권장. *(백엔드 확정 필요)*
- **OQ9. 추천 코스 생성 주체** — 🔶 '기본 추천 코스'를 백엔드가 룰/알고리즘으로 생성하는지, FE가 큐레이션 결과로 조립하는지. 백엔드 생성 권장. *(백엔드 확정 필요)*
