## 0. 화면 설계 프로토타입 (시각 SoT)

화면 설계는 **단일 HTML 인터랙티브 프로토타입**으로 산출되었다. React로 렌더링되며, [DESIGN.md](../DESIGN.md) 디자인 토큰(색·타이포·간격·컴포넌트 패턴)을 그대로 사용한다. 본 문서의 모든 화면 명세는 이 프로토타입을 기준으로 한다.

### 0.1 프로토타입 전역 컨트롤

| 축 | 옵션 | 비고 |
| --- | --- | --- |
| 디바이스 | 모바일(390px) / 데스크톱(1280px) / 둘 다 | 반응형 검수용 동시 미리보기 |
| 테마 | 라이트 / 다크 | `themeStore` light/dark 토글과 1:1 대응 |
| 화면 | S1~S8 (아래 §1) | 좌측 내비게이션으로 전환 |

### 0.2 화면별 레이아웃 변형 (디자인 탐색안)

프로토타입은 핵심 화면마다 복수의 레이아웃 변형을 탐색한다. **각 축의 첫 번째 안이 기본값**이며, 최종 채택안은 구현 시 확정한다.

| 변형 축 | 적용 화면 | 옵션 (기본 ★) |
| --- | --- | --- |
| 히어로(Hero) | S1 메인/검색 | ★검색 중심 / 이미지 히어로 / 미니멀 |
| 플래너 레이아웃 | S2 플래너 | ★3분할 / 지도 중심 / 스택형 |
| 예산 대시보드 | S2 예산 영역 | ★막대 / 도넛 / 카드(분할) |
| POI 카드 | S2 결과/코스 영역 | ★세로형 / 가로형 / 오버레이 |

### 0.3 디자인 시스템 인벤토리 (DESIGN.md 토큰 기반)

프로토타입은 다음 공통 UI 프리미티브를 토큰 기반으로 정의하고 화면에서 재사용한다. 구현 시 `src/components/common`·daisyUI 클래스로 매핑한다.

- 베이스/타이포 유틸/레이아웃 유틸
- 버튼 · 인풋 · 칩/배지 · 카드/패널
- 세그먼트 토글 · 진행 바 · 토스트 · 스켈레톤
- 줄무늬 이미지 플레이스홀더 · 지도 플레이스홀더

> 프로토타입 화면은 실제 콘텐츠 대신 **플레이스홀더(스켈레톤·줄무늬 이미지·지도 박스)** 로 구조를 표현한다. 실제 데이터·카피는 본 문서의 명세 + 백엔드 API 응답으로 채운다.

## 1. 화면 인벤토리 (Information Architecture)

> 프로토타입에서 확정된 화면 목록. `S*`는 화면 식별자, 라우트는 실제 앱 라우트(프로토타입의 데모 경로는 괄호로 병기).

| ID | 화면 | 라우트 | 유형 | 인증 | 핵심 목적 | 레이아웃 변형 |
| --- | --- | --- | --- | --- | --- | --- |
| S1 | 메인 / 검색 | `/` | 페이지(랜딩) | 게스트 | 목적지·일정·인원·테마 입력 → 플래너 진입 | 히어로 3안 |
| S2 | 플래너 워크스페이스 | `/planner` | 페이지 | 게스트(저장 시 로그인) | 결과 탐색 + 코스 빌더 + 실시간 예산 | 레이아웃 3안 · 예산 3안 · POI 카드 3안 |
| S2a | POI 상세 드로어 | (S2 내부 오버레이) | 드로어/시트 | 게스트 | POI 상세 확인 + 코스 추가 | 데스크톱 우측 드로어 / 모바일 바텀시트 |
| S2b | 로그인 모달 | (전역 오버레이) | 모달 | — | 게이팅 시점 카카오 로그인 + 작업 이어하기 | — |
| S3 | 공유 뷰어 | `/share/:id` | 페이지 | 공개(읽기전용) | 수신자가 일정·비용분담 확인, 편집 유도 | — |
| S4 | 컬렉션(마이페이지) | `/collection` | 페이지 | 로그인 | 저장 코스 목록·관리, 프로필 진입점 | — |
| S5 | 로그인 | `/auth/login` | 페이지 | 게스트 | 카카오 OAuth (직접 진입용) | — |
| S6 | 가입 | `/auth/register` | 페이지 | 게스트 | 가입(필요 시) | — |
| S7 | 서비스 소개 | `/about` | 페이지 | 공개 | 서비스 가치·사용법 소개(공모전 심사·온보딩) | — |
| S8 | 404 | `*` | 페이지 | 공개 | NotFound | — |

- **S2a·S2b는 독립 라우트가 아니다.** 프로토타입에서 S2a는 POI 카드/드로어 패턴으로, S2b는 게이팅 시점 모달로 표현된다. 화면 식별자만 부여한다.
- **`/home` 제거**: `/`(메인 검색)와 역할 중복. 라우터·문서에서 제외.

## 2. 화면 상세 명세

> 각 화면의 구성요소·상태(states)·레이아웃 변형·백엔드 요구(데이터/액션)를 정의한다. API 엔드포인트 시그니처는 백엔드가 API 명세서에서 확정(여기선 **요구사항**만 기술).

### S1. 메인 / 검색 (`/`)
- **레이아웃 변형(히어로)**:
  - **검색 중심(기본)**: 히어로 카피 최소화, 한 줄 검색바를 전면에 노출. 빠른 검색 시작 우선.
  - **이미지 히어로**: 경북 대표 이미지(줄무늬 이미지 플레이스홀더) 배경 위 카피 + 검색바. 브랜딩·공모전 첫인상 강조.
  - **미니멀**: 카피·여백 위주, 검색바는 간결. 군더더기 제거.
- **구성요소**: 히어로 카피, 한 줄 검색바 — 목적지(멀티셀렉트, 경북 23개 시군구), 일정(날짜 범위 피커), 인원(카운터), 테마(멀티셀렉트), 원형 검색 버튼.
- **동작**: 검색 → 조건을 URL 쿼리로 직렬화해 `/planner`로 이동(`?dest=…&start=…&end=…&pax=…&theme=…`). 동시에 `searchStore`(Zustand)에 보존.
- **상태**: 기본 / 입력 검증(목적지·일정 미선택 시 안내) / 데이터 미비 목적지 "준비 중" 비활성 표시(OQ3 — 경주·포항·영덕·안동 외).
- **반응형(DESIGN.md §6.2)**: 데스크톱은 가로 한 줄 알약(`sm:rounded-full`), 모바일은 세로 스택(`rounded-2xl`).
- **백엔드 요구**: 시군구 목록·테마 목록(또는 FE 상수). 데이터 보유 시군구 플래그(준비 중 표시용).

### S2. 플래너 워크스페이스 (`/planner`)
- **레이아웃 변형**:
  - **3분할(기본)**: 데스크톱 좌(결과)/우(코스)/하(예산) 3분할. 정보 밀도 최대.
  - **지도 중심**: 지도를 전면 캔버스로, 결과·코스·예산은 오버레이/사이드 패널. 위치 기반 탐색 강조.
  - **스택형**: 섹션을 세로로 쌓음. 모바일 기본이자 데스크톱 저밀도 대안.
- **예산 대시보드 변형**: 막대(기본, 카테고리별 가로 막대) / 도넛(구성비 강조) / 카드(분할, 항목별 카드).
- **POI 카드 변형**: 세로형(기본, 썸네일 위·텍스트 아래) / 가로형(썸네일 좌·텍스트 우, 리스트 밀도↑) / 오버레이(이미지 위 텍스트 오버레이).
- **좌측 — 결과 영역**: 지도(카카오맵, POI 마커) ↔ 리스트 세그먼트 토글. 인원 버킷·테마·시군구 필터 결과. POI 카드(썸네일·이름·카테고리·예상금액).
- **우측 — 내 코스 패널**: `Day1/Day2/…` 탭. Day별 POI 순서 리스트(드래그 정렬), 추가/삭제. 코스 순서대로 지도 경로 표시. 운영시간 충돌 시 경고/선택불가.
- **하단 — 예산 대시보드**: 총예산, 1인당 부담액, 카테고리별(숙박·식비·교통·입장) 소계. **POI별 금액 인라인 수정** → 즉시 재집계. 교통비는 백엔드 추정값 표시(FE 계산 안 함).
- **액션 바**: 저장(→ 게스트면 S2b), 공유(→ S2b 후 링크/카카오), 새 코스.
- **진입**: S1 검색 결과 / `?load=:id`(컬렉션 재편집).
- **상태**: 로딩(스켈레톤) / 결과 없음(빈 상태) / 데이터 준비 중 시군구 / API 오류(재시도) / 저장 성공 토스트.
- **상태 전달**: URL 쿼리 + `searchStore`/`courseStore`/`budgetStore`(Zustand).
- **백엔드 요구**:
  - 검색 조건(시군구·인원버킷·테마·기간)으로 큐레이션된 POI 목록(좌표·카테고리·썸네일·예상 객단가 포함).
  - POI 상세(설명·운영시간·요금·사진·카카오맵 링크) — S2a용.
  - 기본 추천 코스(시군구·인원·일수 기반).
  - 코스 좌표열에 대한 경로/이동거리·교통비 추정.
  - 코스 저장/수정/삭제(로그인 사용자), `load=:id` 조회.

### S2a. POI 상세 드로어 (S2 내부 오버레이)
- **트리거**: 좌측 결과(지도 마커/리스트 카드) 클릭. 라우팅 없이 상태로 제어(데스크톱=우측 드로어, 모바일=바텀시트).
- **구성요소**: 사진, 이름·카테고리, 설명(detailCommon), 운영시간·휴무(detailIntro), 요금/예상 객단가(detailInfo), 카카오맵 링크, **"내 코스(Day N)에 추가"** 버튼.
- **상태**: 로딩 / 상세 오류 / 이미 코스에 포함됨 표시.
- **백엔드 요구**: `contentId` 기반 상세 조회(공통·소개·상세 통합 응답 권장).

### S2b. 로그인 모달 (전역 오버레이)
- **트리거**: 게스트가 저장·공유 클릭 시. 카카오 로그인 버튼.
- **핵심 UX**: 로그인 성공 후 **직전 작업(저장/공유)을 그대로 이어서 수행**. 편집 중 코스 상태는 보존(비로그인 임시 보관 필요 — OQ8).
- **구성요소**: S5 로그인과 동일한 카카오 위젯(`KakoLoginComponent`) 재사용 + 작업 이어하기 안내.
- **백엔드 요구**: 카카오 OAuth 콜백·세션/토큰 발급, 현재 사용자 조회.

### S3. 공유 뷰어 (`/share/:id`)
- **목적**: 링크/카카오톡 수신자가 보는 **읽기전용** 화면. 편집 UI 없음.
- **구성요소**: 여행 요약(목적지·기간·인원), Day별 코스 + 지도 경로(읽기전용), 항목별 금액, **1인당 송금 요망 금액**, "나도 편집하기" CTA(→ 코스 복제하여 `/planner`).
- **인증**: 공개(게스트도 열람). 만료/삭제 시 안내.
- **상태**: 로딩 / 만료·없음(404 유사) / 정상.
- **링크 영속성(OQ5)**: 1단계 임시 URL 인코딩(A) → 2단계 서버 저장(B). `id`는 서버 저장 단계의 식별자. (프로토타입 데모 경로 `/share/demo`)
- **백엔드 요구**: 공유 ID로 코스+예산 스냅샷 조회, 공유 생성(저장 코스 → 공유 ID).

### S4. 컬렉션 / 마이페이지 (`/collection`)
- **목적**: 저장 코스 보관함 + 내 영역(별도 마이페이지 없이 겸함).
- **구성요소**: 저장 코스 카드 그리드(썸네일·제목·기간·인원·총예산). 카드 액션 — 열기(`/planner?load=:id`), 공유(→ S3 링크), 삭제. 헤더 드롭다운에 프로필/로그아웃.
- **상태**: 로딩 / 빈 상태(첫 저장 유도 CTA) / 오류.
- **인증**: 로그인 필요(미로그인 시 S2b 또는 `/auth/login`).
- **백엔드 요구**: 사용자별 저장 코스 목록 조회, 삭제, 코스→공유 링크 생성.

### S5/S6. 인증 (`/auth/login`, `/auth/register`)
- 카카오 OAuth 위젯(`KakoLoginComponent`) 존재. 직접 진입 경로. 게이팅은 주로 S2b 모달로 처리, 본 페이지는 보조.
- `returnTo` 지원(로그인 후 원래 위치 복귀).
- 카카오 버튼은 브랜드 규약 컬러 사용(DESIGN.md §2.2).

### S7. 서비스 소개 (`/about`)
- 서비스 가치·핵심 기능·사용법 소개. 공모전 심사위원·신규 사용자 온보딩용. 정적 콘텐츠.

### S8. 404 (`*`)
- `NotFoundLayout`. 메인으로 복귀 CTA. (프로토타입 데모 경로 `/oops`)

## 3. 화면 전이 흐름 (Flow)

```
[S1 메인/검색] --검색(쿼리)--> [S2 플래너 워크스페이스]
                                  │  ├─ POI 클릭 ──> [S2a 상세 드로어] --추가--> 내 코스(Day N)
                                  │  ├─ 저장/공유(게스트) ──> [S2b 로그인 모달] --성공--> 작업 이어하기
                                  │  ├─ 저장(로그인) ──> [S4 컬렉션]
                                  │  └─ 공유(로그인) ──> 공유 링크/카카오 ──> [S3 공유 뷰어]
[S3 공유 뷰어] --나도 편집하기(코스 복제)--> [S2 플래너]
[S4 컬렉션] --열기(?load=:id)--> [S2 플래너]
[헤더] --> [S1] / [S4] / [S7 소개] / 로그인(S5)
```

## 4. 전역 UX 규칙 (handoff 기준)

- **반응형**: 데스크톱 워크스페이스 ↔ 모바일 탭/시트 전환. 프로토타입 기준 모바일 390px / 데스크톱 1280px, `Layout`의 반응형 폭(최대 1440px) 준수.
- **테마**: 라이트/다크 양쪽에서 검수. 모든 화면은 두 테마에서 깨지지 않아야 한다.
- **상태 패턴**: 모든 데이터 화면은 *로딩(스켈레톤) / 빈 상태 / 오류(재시도)* 3종을 갖춘다.
- **데이터 커버리지(OQ3)**: 목적지는 23개 시군구 멀티셀렉트 전체 노출, 더미 코스·예산 데이터는 경주·포항·영덕·안동 우선. 나머지는 "준비 중" 빈 상태.
- **인증 게이팅(OQ6)**: 탐색·플래닝 게스트 허용, 저장·공유 클릭 시 S2b 모달, 성공 후 작업 이어하기.
- **상태 관리**: 검색조건은 URL 쿼리 + `searchStore`. 코스·예산은 Zustand 스토어(`courseStore`, `budgetStore`). 새로고침·딥링크·공유 견딤.
- **금액 표기**: 원화, 1인당 = 총액 / 실제 `n` 균등 분배(기본).
- **디자인 토큰**: 색·타이포·간격·컴포넌트는 [DESIGN.md](../DESIGN.md) 토큰을 따른다. 임의값(`bg-[#...]`) 도입 시 토큰화 검토.
- **외부 SDK**: 카카오맵(경로·POI), 카카오 공유, 카카오 로그인. 키는 `.env`(노출 금지).
