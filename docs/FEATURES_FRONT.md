# 경북 CoCo — 프론트엔드 기능 분해도 (Feature Breakdown)

> 목적: 요구사항 명세서를 쓰기 **전에**, 구현이 필요한 전체 기능을 화면별로 분해한 상위 문서.
> 여기서는 "무엇이 필요한가"를 기능 단위로 나열하고, 상세 동작 규칙·입출력·검증조건은 다음 단계인 **요구사항 명세서**로 위임한다.
> 범위: 이 저장소는 **FE 전용**(OQ1). 백엔드·DB·관광공사 OpenAPI 수집·가공은 별도 팀 책임이며, FE는 가공된 결과를 자체 백엔드 API로 받아 표시한다.
> 출처: 화면 명세·레이아웃 변형은 [PRD_FRONT.md](PRD_FRONT.md) 및 인터랙티브 화면 설계 프로토타입(`경북 CoCo 화면 설계 (standalone).html`), 공유 비전·핵심기능·통합 미결(OQ)은 [PRD.md](PRD.md), 시각 토큰은 [DESIGN.md](../DESIGN.md)가 단일 출처(SoT). · 버전 0.2 · 기준일 2026-06-06.

---

## 읽는 법

각 기능은 다음 6개 필드를 가진 블록으로 기술한다.

- **설명** — 기능이 하는 일 1~2줄.
- **상태** — 데이터/상호작용 화면이라면 로딩 / 빈(empty) / 에러 상태를 명시.
- **MVP** — ✅ 1차 출시 포함 / 🔜 Post-MVP.
- **API 의존** — FE가 소비할 자체 백엔드 API·데이터(계약 경계). "없음(FE 단독)"이면 백엔드 불필요.
- **모듈** — 구현 위치 힌트 (라우트 / 스토어 / 주요 컴포넌트).
- **가치** — 왜 필요한가. PRD 문제(P1 소규모 니즈 / P2 예산 예측 / P3 계획 피로감)와 연결.

기능 ID 규약: 공통 `C#`, 디자인 시스템 `D#`, 메인 검색 `I#`, 플래너 `P#`, 컬렉션 `CO#`, 인증 `A#`.

화면 집합(프로토타입 기준): **공통 · 디자인 시스템 · 메인 검색(`/`) · 플래너(`/planner`) · 컬렉션(`/collection`) · 공유(`/share/:id`) · 인증(`/auth/*`) · 소개(`/about`)**.

## 0. 공통 (Cross-cutting)

여러 화면에서 동일하게 동작하므로 화면 섹션에서 빼낸 기능들.

### C1. 전역 셸 레이아웃
- **설명**: Header + main + Footer 구조와 반응형 폭(최대 1440px)을 모든 페이지에 공통 적용.
- **상태**: 해당 없음.
- **MVP**: ✅ (이미 구현 — `Layout.tsx`)
- **API 의존**: 없음(FE 단독).
- **모듈**: `components/layout/Layout.tsx`, `HeaderLayout.tsx`, `FooterLayout.tsx`.
- **가치**: 일관된 탐색 경험.

### C2. 글로벌 네비게이션
- **설명**: 헤더 메뉴(로고→메인, 플래너, 컬렉션, 로그인/프로필 진입), 현재 위치 표시. 데스크톱 가로 탭 ↔ 모바일 햄버거 드롭다운.
- **상태**: 비로그인/로그인에 따라 헤더 우측 액션(로그인 ↔ 프로필/로그아웃) 분기.
- **MVP**: ✅
- **API 의존**: 없음(FE 단독, 로그인 상태는 C5 참조).
- **모듈**: `HeaderLayout.tsx`.
- **가치**: 화면 간 이동 동선 확보.

### C3. 테마 토글 (라이트/다크)
- **설명**: light/dark 테마 전환, `localStorage` 영속 + `data-theme` 동기화. 모든 화면을 양쪽 테마에서 검수.
- **상태**: 해당 없음.
- **MVP**: ✅ (이미 구현 — `themeStore`, `ThemeController`)
- **API 의존**: 없음(FE 단독).
- **모듈**: `stores/themeStore.ts`, `components/common/ThemeController.tsx`.
- **가치**: 사용자 편의. 프로토타입이 라이트/다크 양안을 전제로 설계됨.

### C4. 인증 게이팅 / 로그인 유도 (하이브리드)
- **설명**: 탐색·플래닝은 게스트 허용, **저장·공유 시점**에만 로그인 모달로 유도(OQ6 하이브리드).
- **상태**: 로그인됨(통과) / 비로그인(모달 노출) / 모달 취소(작업 보류).
- **MVP**: ✅
- **API 의존**: 없음(FE 게이팅 로직). 로그인 자체는 C5.
- **모듈**: `authStore`(신규), 로그인 유도 모달 컴포넌트(S2b), 각 저장/공유 액션에서 호출.
- **가치**: P3 — 진입 마찰 최소화하면서 핵심 행동(저장/공유) 시점에만 전환 요구.

### C5. 카카오 OAuth 로그인 세션
- **설명**: 카카오 로그인으로 발급된 토큰·사용자 식별 정보를 앱 전역 세션으로 보관·복원, 로그아웃 시 폐기.
- **상태**: 로딩(인증 진행) / 로그인됨 / 비로그인 / 인증 실패.
- **MVP**: ✅ (위젯 일부 구현 — `KakoLoginComponent`. 현재 토큰을 `localStorage`에 저장 후 `/`로 이동하는 수준이라 세션 스토어로 승격 필요)
- **API 의존**: (단계적) 자체 백엔드의 카카오 토큰 검증/세션 발급 API — 초기엔 FE 토큰 보관으로 시작 가능.
- **모듈**: `authStore`(신규), `KakoLoginComponent.tsx`, `api/`(axios 인스턴스).
- **가치**: 저장·공유·개인화의 전제.

### C6. 공유 (카카오톡 / 링크)
- **설명**: 완성 코스 + 상세 예산 내역(항목별 금액, 1인당 송금 요망액)을 카카오톡 공유 또는 링크 복사로 전달(F3).
- **상태**: 공유 준비 / 공유 시트 호출 / 링크 복사 완료 토스트 / 공유 실패.
- **MVP**: ✅
- **API 의존**: (단계적, OQ5) **A. 임시 URL 인코딩**(공유 상태를 링크 파라미터에 직렬화, 백엔드 불필요) → **B. 서버 저장 공유 API**(영속 단축 링크)로 전환.
- **모듈**: 공유 액션 컴포넌트(공통), 카카오 공유 SDK 래퍼, 플래너·컬렉션에서 호출.
- **가치**: P3 — 동행인과의 코스·더치페이 공유 마찰 제거. 차별점(통합 공유).

### C7. 상태 패턴 (로딩 / 빈 / 에러)
- **설명**: 데이터 패칭 화면이 공유하는 로딩 스켈레톤·스피너, 빈 상태 안내, 에러+재시도 UI 규약.
- **상태**: (본 항목이 정의하는 대상)
- **MVP**: ✅ (로딩 일부 구현 — `Loading.tsx`)
- **API 의존**: 없음(FE 단독).
- **모듈**: `components/common/Loading.tsx`, 공통 EmptyState/ErrorState/Skeleton 컴포넌트(신규).
- **가치**: 일관된 피드백으로 신뢰감·체감 성능 향상.

### C8. 토스트 / 알림
- **설명**: 저장 성공, 링크 복사, 에러 등 비차단 피드백 표시.
- **상태**: 해당 없음.
- **MVP**: ✅
- **API 의존**: 없음(FE 단독).
- **모듈**: 공통 토스트 컴포넌트/훅(신규, DESIGN 토스트 토큰 기반).
- **가치**: 행동 결과의 즉시 피드백.

### C9. 404 / 에러 라우트
- **설명**: 미존재 경로·라우트 에러 처리 화면(S8).
- **상태**: 해당 없음.
- **MVP**: ✅ (이미 구현 — `NotFoundLayout`)
- **API 의존**: 없음(FE 단독).
- **모듈**: `components/layout/NotFoundLayout.tsx`, `router.tsx`의 `errorElement`.
- **가치**: 잘못된 진입의 회복 동선.

### C10. API 클라이언트 기반
- **설명**: axios 인스턴스(기본 URL·헤더·토큰 주입·에러 정규화 인터셉터) 등 데이터 패칭 공통 토대.
- **상태**: 해당 없음(인프라).
- **MVP**: ✅ (미구현 — `api/` 비어 있음)
- **API 의존**: 자체 백엔드 베이스 URL(환경변수 `VITE_` 접두사).
- **모듈**: `api/`(신규), 커스텀 훅 `hooks/`.
- **가치**: 모든 데이터 기능의 전제 인프라.

## 0.5 디자인 시스템 (Design System)

프로토타입은 [DESIGN.md](../DESIGN.md) 토큰을 기반으로 공통 UI 프리미티브를 정의한다. 화면 기능과 별개로 **재사용 컴포넌트 라이브러리**로 구현한다.

### D1. 디자인 토큰 정합
- **설명**: 색·타이포·간격을 `src/index.css`의 `@theme`/daisyUI 시맨틱으로 일원화. 프로토타입의 토큰/베이스/타이포 유틸/레이아웃 유틸과 1:1 정합.
- **상태**: 해당 없음.
- **MVP**: ✅ (대부분 구현 — `index.css`)
- **API 의존**: 없음(FE 단독).
- **모듈**: `src/index.css`.
- **가치**: 일관된 비주얼, 임의값 최소화.

### D2. 공통 컴포넌트 키트
- **설명**: 버튼 · 인풋 · 칩/배지 · 카드/패널 · 세그먼트 토글 · 진행 바 · 토스트 · 스켈레톤 · 이미지/지도 플레이스홀더를 재사용 컴포넌트로 제공.
- **상태**: 컴포넌트별 기본/비활성/에러 등 변형.
- **MVP**: ✅ (일부만 구현)
- **API 의존**: 없음(FE 단독).
- **모듈**: `components/common/*`(신규 다수), daisyUI 클래스.
- **가치**: 화면 구현 속도·일관성. 프로토타입과 동일한 시각 산출.

### D3. POI 카드 변형
- **설명**: POI 카드를 **세로형(기본)/가로형/오버레이** 3변형으로 제공. 결과 리스트·코스 패널·컬렉션에서 컨텍스트별 선택.
- **상태**: 로딩(스켈레톤) / 이미지 없음(플레이스홀더) / 코스 포함됨 배지.
- **MVP**: ✅ (변형 중 채택안 1개 우선, 나머지 점진)
- **API 의존**: POI 썸네일·이름·카테고리·예상금액(P2 응답).
- **모듈**: `components/planner/PoiCard.tsx`(신규).
- **가치**: P1/P3 — 탐색 밀도와 시각적 매력의 균형.

### D4. 예산 대시보드 시각화 변형
- **설명**: 예산 대시보드를 **막대(기본)/도넛/카드** 3변형으로 제공. 총예산·1인당·카테고리 소계를 시각화.
- **상태**: 코스 비었을 때 0원 / 산출 표시 / 데이터 누락 항목 표기.
- **MVP**: ✅ (채택안 1개 우선)
- **API 의존**: 장소별 예산 기본값·교통비(P9/P11 메타데이터). 합산은 FE.
- **모듈**: `components/planner/BudgetDashboard.tsx`(신규).
- **가치**: P2 — 예산을 직관적으로 전달(핵심 차별점).

## 1. 메인 검색 — `/` (`Index.tsx`)

F1(인원수 기반 큐레이션)의 진입점. 검색 조건을 모아 플래너로 넘긴다. 검색 UI는 구현돼 있으나 옵션 데이터·검색 실행·상태 연결이 미구현.

### I0. 히어로 변형
- **설명**: 메인 히어로를 **검색 중심(기본)/이미지 히어로/미니멀** 중 채택안으로 구현. 검색바를 전면 노출하거나 브랜딩 이미지 위에 배치.
- **상태**: 일반.
- **MVP**: ✅ (채택안 1개 우선)
- **API 의존**: 없음(FE 단독). 이미지 히어로 시 정적 에셋.
- **모듈**: `Index.tsx`, 히어로 컴포넌트.
- **가치**: P3 — 첫인상·공모전 심사 인상. 빠른 검색 시작.

### I1. 목적지 멀티셀렉트
- **설명**: 경북 23개 시군구 멀티셀렉트(OQ3). 더미 코스·예산 데이터는 **경주·포항·영덕·안동 4곳 우선**, 나머지는 "준비 중" 빈 상태.
- **상태**: 일반 / "준비 중" 지역 선택 시 안내.
- **MVP**: ✅ (현재 옵션이 더미값 → 시군구 목록으로 교체 필요)
- **API 의존**: 시군구 목록은 정적 상수로 가능. 지역별 데이터 가용 여부는 플래너 단계 API에 의존.
- **모듈**: `Index.tsx`, 지역 상수, `searchStore`(신규).
- **가치**: P1 — 경북 내 지역 타게팅.

### I2. 일정(기간) 선택
- **설명**: 시작·종료일 range 선택. 일수는 예산·코스 분배의 기준.
- **상태**: 미선택 / 선택됨.
- **MVP**: ✅ (이미 구현 — `react-datepicker` range)
- **API 의존**: 없음(FE 단독).
- **모듈**: `Index.tsx`, `searchStore`.
- **가치**: P2 — 일정 길이가 예산·코스 규모를 좌우.

### I3. 인원 카운터
- **설명**: 인원수 입력(카운터, `number`). 추천 시 버킷 매핑(`1→1`, `2→2`, `≥3→'3-4'`), 1인당 예산 분배는 실제 `n` 사용(OQ2).
- **상태**: 최소 1명 보장.
- **MVP**: ✅ (카운터 구현됨, 버킷 매핑 로직 미구현)
- **API 의존**: 버킷은 큐레이션 파라미터로 백엔드 POI 조회에 전달.
- **모듈**: `Index.tsx`, `searchStore`, 버킷 매핑 유틸.
- **가치**: P1 — 인원수 기반 큐레이션의 핵심 입력.

### I4. 테마 멀티셀렉트
- **설명**: 지역 무관 일반 테마(어드벤처/휴식/문화/음식 등) 복수 선택, 확장 가능(OQ3).
- **상태**: 미선택(전체) / 선택됨.
- **MVP**: ✅ (이미 구현)
- **API 의존**: 테마는 큐레이션 필터 파라미터로 전달.
- **모듈**: `Index.tsx`, `searchStore`.
- **가치**: P1 — 취향 기반 큐레이션.

### I5. 검색 실행 → 플래너 이동
- **설명**: 선택한 목적지·일정·인원·테마를 확정해 `/planner`로 전달(URL 쿼리 + 스토어).
- **상태**: 필수값 미충족 시 비활성/안내 / 실행.

- **MVP**: ✅ (라우트·페이지 존재 — `Register.tsx`. 흐름 정의 필요. 카카오 단일 방식이면 별도 가입이 최소화될 수 있음 → 명세서에서 확정)
- **API 의존**: **가입/사용자 등록 API**.
- **모듈**: `pages/Auth/Register.tsx`, `authStore`.
- **가치**: 사용자 식별·개인화 시작점.

### A3. 로그인 콜백 / 리다이렉트 처리
- **설명**: OAuth 성공/실패 후 라우팅, 에러 시 로그인 화면 복귀.
- **상태**: 처리 중 / 성공 이동 / 실패 안내.
- **MVP**: ✅ (현재 성공 시 `/`로 `replace` 이동만)
- **API 의존**: 없음(FE 라우팅). 토큰 검증은 C5/A1.
- **모듈**: `KakoLoginComponent.tsx`, `authRouter.tsx`.
- **가치**: 인증 흐름 완결.

### A4. 로그아웃
- **설명**: 세션·토큰 폐기 후 게스트 상태로 전환.
- **상태**: 로그아웃 / 완료.
- **MVP**: ✅ (미구현)
- **API 의존**: (선택) 백엔드 세션 무효화 API.
- **모듈**: `authStore`, `HeaderLayout.tsx`.
- **가치**: 계정 보호·전환.

### A5. 로그인 후 원래 위치 복귀 (returnTo)
- **설명**: C4 게이팅으로 로그인한 경우, 인증 후 직전 작업(저장/공유) 위치로 복귀.
- **상태**: 복귀 대상 있음 / 없음(기본 이동).
- **MVP**: ✅
- **API 의존**: 없음(FE 라우팅 상태).
- **모듈**: `authStore`/라우팅 상태, C4 모달(S2b).
- **가치**: P3 — 인증 때문에 작업 흐름이 끊기지 않게.
