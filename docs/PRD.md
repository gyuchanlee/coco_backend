# 경북 CoCo — 제품 기획서 (PRD)

> 출처: 『2026 관광데이터 활용 공모전』 웹·앱 개발 부문 제안서.
> 이 문서는 제안서의 제품 비전을 repo 기준 명세로 옮기고, 현재 코드베이스 구조에 매핑한 것이다.
> 백엔드·DB 등 아직 확정되지 않은 부분은 [§10 가정 및 미결](#10-가정-및-미결-open-questions)로 분리한다.
> 버전 0.1 · 기준일 2026-05-30.

## 1. 한 줄 정의

**경북 지역 소규모 여행자(1~4인)를 위한 추천 & 예산 스마트 플래너.**

인원수에 맞춘 큐레이션으로 숙소·식당·관광지를 추천하고, 코스를 구성하는 즉시 총예산과 1인당 부담액을 실시간 산출하며, 완성된 일정과 비용 분담 내역을 카카오톡으로 한 번에 공유한다.

## 2. 배경 및 문제

| # | 문제 | 근거 |
| --- | --- | --- |
| P1 | **소규모 여행 니즈 미충족** | 1인 가구 증가·개인화로 혼행/2~3인 소그룹이 주류가 되었으나, 기존 플랫폼은 4인 이상·가족 단위 정보에 편중. |
| P2 | **예산 예측의 번거로움** | 고물가·가성비 중심 소비. 숙박·식비·교통·입장료를 일일이 조사·합산해 예산을 예측하기 어렵다. |
| P3 | **여행 계획 피로감** | 블로그·SNS·예약앱·지도앱을 오가며 정보를 교차 검증하고, 동행인과 코스 공유·더치페이를 논의하는 과정의 마찰. |

## 3. 타깃 사용자 & 핵심 시나리오

- **주 타깃**: 가성비·계획 효율을 중시하는 2030 세대 소규모 여행자(1~4인).
- **핵심 시나리오**
  1. 인원·일정·목적지·테마를 한 줄로 선택해 검색한다. *(메인 검색 UI)*
  2. 인원수에 맞게 큐레이션된 스팟을 지도/리스트로 보고, 우측 패널의 기본 추천 코스를 커스텀하거나 새 코스를 만든다.
  3. 코스에 스팟을 추가/삭제할 때마다 하단 대시보드의 총예산·1인당 금액이 실시간 갱신된다.
  4. 완성된 코스와 비용 분담 내역을 카카오톡/링크로 동행인에게 공유한다.
  5. 마음에 든 코스를 컬렉션에 저장한다.

## 4. 핵심 기능

제안서의 3대 기능 축을 그대로 채택한다. 각 기능을 현재 라우트/페이지/스토어/API 계층에 매핑한다 (상세 매핑은 [§9](#9-구현-매핑-코드베이스--기능)).

### F1. Who & How Many — 인원수 기반 스마트 큐레이션
- 여행 인원(**1인 / 2인 / 3~4인**)과 일정을 선택하면 해당 인원수에 적합한 경북 내 숙소·식당·관광지가 맞춤 노출.
- 예: 혼자 먹기 좋은 바테이블 식당(1인), 감성 숙소(2인), 비용 나누기 좋은 패밀리룸(3~4인).
- **연결 화면**: 메인 검색(`/`, `Index.tsx`) → 결과/플래너(`/planner`).

### F2. Smart Itinerary & Budget Planner — 실시간 예산 계산기 & 코스 빌더
- **결과 표시**: 여행 스팟을 지도 / 리스트 중 선택해 확인.
- **기본 추천 코스**: 우측 패널에 기본 일정 표시, 커스텀 또는 새 코스 생성.
- **코스 빌더**: 지도에서 POI(Point of Interest)를 클릭해 '내 코스'에 추가/삭제.
- **시간대 반영**: 박물관·식당 등 운영 시간을 반영, 불가능한 시간대 코스는 선택 불가 표시.
- **실시간 예산 계산**: 장소별 평균 객단가(숙박비·식사비·입장료)를 기반으로 총예상 예산과 1인당 부담액을 하단 대시보드에서 실시간 업데이트.
- **경로 시각화**: 코스 순서대로 이동 경로·지도 자동 표시 (카카오지도 API로 경로·POI 시각화).
- **연결 화면**: 플래너(`/planner`, `Planner.tsx`).

### F3. One-Click Share — 카카오톡 기반 비용 분담 공유
- 완성된 여행 일정 + 상세 예산 내역(항목별 금액, 1인당 송금 요망 금액)을 카카오톡 또는 링크로 즉시 공유.
- 카카오톡 공유하기 API로 간단 공유.
- **연결 화면**: 플래너 결과 공유 액션 / 컬렉션(`/collection`).

## 5. 차별성

| 구분 | 기존 여행 서비스 (포털·OTA) | 경북 CoCo (스마트 플래너) |
| --- | --- | --- |
| 추천 기준 | 평점·리뷰·광고 위주 | **여행 인원수 및 1인당 예산 최적화** 위주 |
| 비용 계산 | 개별 상품(숙소/티켓) 결제 가격만 | **숙식·교통 포함 전체 여정 예산 시뮬레이션** |
| 공유 방식 | 개별 링크 복사 후 전송 | **완성 코스 + 비용분담 내역 통합 공유** |

## 6. 데이터 활용 (한국관광공사 OpenAPI — 필수)

활용 API: **국문 관광정보 조회 서비스** — `areaBasedList`(지역기반 조회), `detailCommon`(공통정보), `detailIntro`(소개정보), `detailInfo`(상세정보).

> 아래 수집·가공은 모두 **백엔드 책임**이다([ADR 0007](./adr/0007-backend-db-ownership.md)). 이 FE repo는 가공된 결과를 백엔드 API로 받아 표시한다.

| 단계 | 방식 |
| --- | --- |
| ① 지역 필터링·기초 수집 | `areaBasedList`에 `areaCode=35`(경상북도) + `sigunguCode`를 파라미터로, contentTypeId별 수집: 관광지(12)·문화시설(14)·숙박(32)·음식점(39). 안정적 응답을 위해 서비스 DB에 적재, **정합성 위해 월 1회 주기 수집**. |
| ② 인원·예산 메타데이터 생성 | 수집한 POI의 콘텐츠 ID로 상세정보 API 호출. **음식점**: 외식통계조회·한국관광 데이터랩 '지역별 관광소비' 데이터로 평균 식비 근사치 제공, 카카오맵 링크로 사용자 수정 가능. **숙박**: 객실 수용 인원·비수기/성수기 요금 분석 → 1·2·4인 적합 숙소 분류, 예산 계산기 기본값. |
| ③ 동선 최적화·교통비 추정 | DB의 `MapX`/`MapY` 좌표로 선택 장소 간 직선/예상 이동거리 계산 → 대략적 유류비·대중교통비를 예산에 자동 합산. |

## 7. 지역 특화 테마 (경북)

- **[경주]** 나홀로 도보 역사 산책 & 황리단길 혼밥 투어
- **[포항/영덕]** 2~3인 해안도로 드라이브 & 오션뷰 가성비 숙소
- **[안동]** 4인 친구 모임 전통 한옥 고택 체험 & 찜닭 거리 투어
- **로컬 상권 연계**: 소규모 식당·카페 정보 제공으로 지역 경제 선순환 유도.

## 8. 발전 방향 & 기대효과

**발전 단계**
- **[단기] UGC 데이터 축적** — 공공 API 의존에서 시작, 사용자가 만든 '가성비 실전 코스'·'실제 지출 비용'을 리뷰로 축적해 추천 로직 정확도 고도화.
- **[중기] 소상공인 제휴 할인** — 추천 골목 상권 식당·카페와 제휴, 앱 내 전용 쿠폰 발급 + 수수료 기반 BM.
- **[장기] B2B/B2G 확장** — 경북 실증 모델을 전국 확장, '소규모 여행객 이동·소비 동선 데이터'를 비식별화해 지자체 관광 정책용으로 제공.

**기대효과**
- *경제적*: 예산 투명성으로 2030 경북 관광 진입 장벽 완화, 소비를 골목상권으로 분산.
- *사회적*: 계획·정산 마찰과 피로감 감소 → 만족도 높은 여행 경험.
- *공공데이터 가치*: 단순 관광지 정보를 '실시간 예산 자동 계산' 유틸리티와 결합해 고부가가치 정보로 재창출.

## 9. 구현 매핑 (코드베이스 ↔ 기능)

현재 라우트/모듈은 셸 수준이며, 기능 로직은 대부분 미구현이다.

| 기능 | 라우트 | 파일 | 현재 상태 |
| --- | --- | --- | --- |
| 메인 검색(F1 진입) | `/` | `src/pages/Index.tsx` | 검색 UI(목적지·일정·인원·테마) 구현됨. 단, 인원은 자유 카운터 — [OQ2](#10-가정-및-미결-open-questions) 참고. |
| 코스 빌더 + 예산(F2) | `/planner` | `src/pages/Planner/Planner.tsx` | 플레이스홀더. 지도·코스·예산 로직 미구현. |
| 비용 분담 공유(F3) | `/planner`·`/collection` | `src/pages/Collection/Collection.tsx` | 플레이스홀더. |
| 인증 | `/auth/login`·`/auth/register` | `src/pages/Auth/*`, `src/components/auth/KakoLoginComponent.tsx` | 카카오 OAuth 위젯 존재. |
| API 계층 | — | `src/api/` | **비어 있음.** 자체 백엔드 API 클라이언트(+ 카카오 지도/공유 SDK) 미작성. 관광공사 OpenAPI는 백엔드가 담당 — [ADR 0007](./adr/0007-backend-db-ownership.md). |
| 상태 관리 | — | `src/stores/` | `themeStore`만 존재. 검색 조건·코스·예산 스토어 미작성. |
| 커스텀 훅 | — | `src/hooks/` | 비어 있음. |

## 10. 가정 및 미결 (Open Questions)

> 제안서에 명시되지 않았거나 코드베이스와 어긋나는 사항. 구현 착수 전 확정 필요.
> 아키텍처급 쟁점(OQ1·OQ2·OQ5·OQ6)은 ADR로 추적한다(상태 `Proposed`).

- **OQ1. 백엔드/DB 소유 주체** — ✅ **확정**: 백엔드·DB·수집은 별도 repo/팀, 이 repo는 FE 전용. `src/api/`는 자체 백엔드 API만 소비(관광공사 직접 호출 안 함). → [ADR 0007 (Accepted)](./adr/0007-backend-db-ownership.md)
- **OQ2. 인원 입력 모델 불일치** — 제안서는 1/2/3~4인 **버킷** 큐레이션을 요구하나, 현재 `Index.tsx`는 자유 증감 카운터(`number`). 버킷으로 변경할지, 카운터를 버킷에 매핑할지 결정 필요. → [ADR 0008](./adr/0008-party-size-input-model.md)
- **OQ3. 목적지·테마 옵션 더미 데이터** — `Index.tsx`의 목적지(Inter/Poppins/Raleway)·테마 옵션은 플레이스홀더. 경북 시군구·실제 테마로 교체 필요. *(구현 TODO — ADR 비대상)*
- **OQ4. 예산 데이터 출처 신뢰도** — 식비·교통비 근사치의 초기 정확도 한계. 추가 통계자료 확보 전까지 사용자 수정 가능으로 보완(제안서 명시). *(데이터 소싱 정책 — ADR 비대상)*
- **OQ5. 공유 링크 영속성** — 카카오톡/링크 공유 시 코스 데이터를 어디에 저장·조회할지(서버 저장 vs URL 인코딩) 미정. → [ADR 0009](./adr/0009-shared-course-persistence.md)
- **OQ6. 인증 게이팅** — 컬렉션 저장·공유에 로그인 필수 여부 미정. → [ADR 0010](./adr/0010-auth-gating-policy.md)
